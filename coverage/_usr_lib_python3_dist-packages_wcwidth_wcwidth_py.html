18In fixed-width output devices, Latin characters all occupy a single 

19"cell" position of equal width, whereas ideographic CJK characters 

20occupy two such cells. Interoperability between terminal-line 

21applications and (teletype-style) character terminals using the 

22UTF-8 encoding requires agreement on which character should advance 

23the cursor by how many cell positions. No established formal 

24standards exist at present on which Unicode character shall occupy 

25how many cell positions on character terminals. These routines are 

26a first attempt of defining such behavior based on simple rules 

27applied to data provided by the Unicode Consortium. 

28 

29For some graphical characters, the Unicode standard explicitly 

30defines a character-cell width via the definition of the East Asian 

31FullWidth (F), Wide (W), Half-width (H), and Narrow (Na) classes. 

32In all these cases, there is no ambiguity about which width a 

33terminal shall use. For characters in the East Asian Ambiguous (A) 

34class, the width choice depends purely on a preference of backward 

35compatibility with either historic CJK or Western practice. 

36Choosing single-width for these characters is easy to justify as 

37the appropriate long-term solution, as the CJK practice of 

38displaying these characters as double-width comes from historic 

39implementation simplicity (8-bit encoded characters were displayed 

40single-width and 16-bit ones double-width, even for Greek, 

41Cyrillic, etc.) and not any typographic considerations. 

42 

43Much less clear is the choice of width for the Not East Asian 

44(Neutral) class. Existing practice does not dictate a width for any 

45of these characters. It would nevertheless make sense 

46typographically to allocate two character cells to characters such 

47as for instance EM SPACE or VOLUME INTEGRAL, which cannot be 

48represented adequately with a single-width glyph. The following 

49routines at present merely assign a single-cell width to all 

50neutral characters, in the interest of simplicity. This is not 

51entirely satisfactory and should be reconsidered before 

52establishing a formal standard in this area. At the moment, the 

53decision which Not East Asian (Neutral) characters should be 

54represented by double-width glyphs cannot yet be answered by 

55applying a simple rule from the Unicode database content. Setting 

56up a proper standard for the behavior of UTF-8 character terminals 

57will require a careful analysis not only of each Unicode character, 

58but also of each presentation form, something the author of these 

59routines has avoided to do so far.

77def _bisearch(ucs, table): 

78 """ 

79 Auxiliary function for binary search in interval table. 

80 

81 :arg int ucs: Ordinal value of unicode character. 

82 :arg list table: List of starting and ending ranges of ordinal values, 

83 in form of ``[(start, end), ...]``. 

84 :rtype: int 

85 :returns: 1 if ordinal value ucs is found within lookup table, else 0. 

86 """ 

87 lbound = 0 

88 ubound = len(table) - 1 

89 

90 if ucs < table[0][0] or ucs > table[ubound][1]: 

91 return 0 

92 while ubound >= lbound: 

93 mid = (lbound + ubound) // 2 

94 if ucs > table[mid][1]: 

95 lbound = mid + 1 

96 elif ucs < table[mid][0]: 

97 ubound = mid - 1 

98 else: 

99 return 1 

100 

101 return 0

104def wcwidth(wc): 

105 r""" 

106 Given one unicode character, return its printable length on a terminal. 

107 

108 The wcwidth() function returns 0 if the wc argument has no printable effect 

109 on a terminal (such as NUL '\0'), -1 if wc is not printable, or has an 

110 indeterminate effect on the terminal, such as a control character. 

111 Otherwise, the number of column positions the character occupies on a 

112 graphic terminal (1 or 2) is returned. 

113 

114 The following have a column width of -1: 

115 

116 - C0 control characters (U+001 through U+01F). 

117 

118 - C1 control characters and DEL (U+07F through U+0A0). 

119 

120 The following have a column width of 0: 

121 

122 - Non-spacing and enclosing combining characters (general 

123 category code Mn or Me in the Unicode database). 

124 

125 - NULL (U+0000, 0). 

126 

127 - COMBINING GRAPHEME JOINER (U+034F). 

128 

129 - ZERO WIDTH SPACE (U+200B) through 

130 RIGHT-TO-LEFT MARK (U+200F). 

131 

132 - LINE SEPERATOR (U+2028) and 

133 PARAGRAPH SEPERATOR (U+2029). 

134 

135 - LEFT-TO-RIGHT EMBEDDING (U+202A) through 

136 RIGHT-TO-LEFT OVERRIDE (U+202E). 

137 

138 - WORD JOINER (U+2060) through 

139 INVISIBLE SEPARATOR (U+2063). 

140 

141 The following have a column width of 1: 

142 

143 - SOFT HYPHEN (U+00AD) has a column width of 1. 

144 

145 - All remaining characters (including all printable 

146 ISO 8859-1 and WGL4 characters, Unicode control characters, 

147 etc.) have a column width of 1. 

148 

149 The following have a column width of 2: 

150 

151 - Spacing characters in the East Asian Wide (W) or East Asian 

152 Full-width (F) category as defined in Unicode Technical 

153 Report #11 have a column width of 2. 

154 """ 

155 # pylint: disable=C0103 

156 # Invalid argument name "wc" 

157 ucs = ord(wc) 

158 

159 # NOTE: created by hand, there isn't anything identifiable other than 

160 # general Cf category code to identify these, and some characters in Cf 

161 # category code are of non-zero width. 

162 

163 # pylint: disable=too-many-boolean-expressions 

164 # Too many boolean expressions in if statement (7/5) 

165 if (ucs == 0 or 

166 ucs == 0x034F or 

167 0x200B <= ucs <= 0x200F or 

168 ucs == 0x2028 or 

169 ucs == 0x2029 or 

170 0x202A <= ucs <= 0x202E or 

171 0x2060 <= ucs <= 0x2063): 

172 return 0 

173 

174 # C0/C1 control characters 

175 if ucs < 32 or 0x07F <= ucs < 0x0A0: 

176 return -1 

177 

178 # combining characters with zero width 

179 if _bisearch(ucs, ZERO_WIDTH): 

180 return 0 

181 

182 return 1 + _bisearch(ucs, WIDE_EASTASIAN)

185def wcswidth(pwcs, n=None): 

186 """ 

187 Given a unicode string, return its printable length on a terminal. 

188 

189 Return the width, in cells, necessary to display the first ``n`` 

190 characters of the unicode string ``pwcs``. When ``n`` is None (default), 

191 return the length of the entire string. 

192 

193 Returns ``-1`` if a non-printable character is encountered. 

194 """ 

195 # pylint: disable=C0103 

196 # Invalid argument name "n" 

197 

198 end = len(pwcs) if n is None else n 

199 idx = slice(0, end) 

200 width = 0 

201 for char in pwcs[idx]: 

202 wcw = wcwidth(char) 

203 if wcw < 0: 

204 return -1 

205 else: 

206 width += wcw 

207 return width
