1""" 

2Jedi is a static analysis tool for Python that can be used in IDEs/editors. 

3Jedi has a focus on autocompletion and goto functionality. Jedi is fast and is 

4very well tested. It understands Python and stubs on a deep level. 

5 

6Jedi has support for different goto functions. It's possible to search for 

7usages and list names in a Python file to get information about them. 

8 

9Jedi uses a very simple API to connect with IDE's. There's a reference 

10implementation as a `VIM-Plugin <https://github.com/davidhalter/jedi-vim>`_, 

11which uses Jedi's autocompletion. We encourage you to use Jedi in your IDEs. 

12Autocompletion in your REPL is also possible, IPython uses it natively and for 

13the CPython REPL you have to install it. 

14 

15Here's a simple example of the autocompletion feature: 

16 

17>>> import jedi 

18>>> source = ''' 

19... import json 

20... json.lo''' 

21>>> script = jedi.Script(source, 3, len('json.lo'), 'example.py') 

22>>> script 

23<Script: 'example.py' ...> 

24>>> completions = script.completions() 

25>>> completions 

26[<Completion: load>, <Completion: loads>] 

27>>> print(completions[0].complete) 

28ad 

29>>> print(completions[0].name) 

30load 

31 

32As you see Jedi is pretty simple and allows you to concentrate on writing a 

33good text editor, while still having very good IDE features for Python. 

34"""
